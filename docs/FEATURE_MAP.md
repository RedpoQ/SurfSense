## 详细说明

### 1. 资料输入

- **用户价值：** 上传中文教材 PDF、学术论文、课堂笔记、网页剪藏等资料，构建个人知识库。
- **SurfSense 基础能力：** 已支持 50+ 文件格式上传，包括 PDF/DOCX/EPUB/图片/音频/视频。已有文档上传 API 和前端 UI。
- **是否配置即可：** 是。Docker 部署后即可使用。
- **是否需要改造：** 低。主要改造点在解析质量（见模块 2）。
- **是否需要新增：** 否。
- **前置依赖：** Docker 环境、ETL_SERVICE 配置。
- **当前是否允许开发：** Phase 0 不允许，Phase 1 允许配置。

### 2. 中文资料解析优化

- **用户价值：** 中文 PDF（含公式、表格、分栏排版）能被正确解析为结构化 Markdown，不丢失内容。
- **SurfSense 基础能力：** 已有 ETL 管道（DOCLING / UNSTRUCTURED / LLAMACLOUD），但默认配置偏向英文。
- **是否配置即可：** 部分。DOCLING 支持中文但效果需实测；UNSTRUCTURED 和 LLAMACLOUD 对中文 PDF 的处理质量未知。
- **是否需要改造：** 可能需要：
  - 调整 DOCLING 的中文 OCR 参数
  - 针对教材场景的分栏/公式/图表特殊处理
  - 中文分词对 chunking 质量的影响（Chonkie 默认用空格/标点切分）
- **是否需要新增：** 可能需要新增中文教材专用解析器。
- **前置依赖：** Docker 环境、ETL_SERVICE 选择、中文 PDF 测试样本。
- **当前是否允许开发：** 不允许。需要先完成 Phase 1 的 Docker 启动验证和 ETL 服务评估。

### 3. 中文问答

- **用户价值：** 用中文提问，获得基于中文资料的引用回答。
- **SurfSense 基础能力：** 已有 RAG 问答链路（混合搜索 + LiteLLM + Agent 架构）。已有中文 system prompt 示例（DeepSeek 配置中）。
- **是否配置即可：** 部分。配置 DeepSeek/Qwen API key 后 LLM 可用；但中文搜索质量取决于中文 embedding。
- **是否需要改造：** 需要：
  - 中文化 Agent 系统提示词
  - 中文化引用格式
  - 中文 chunk 检索质量优化
- **是否需要新增：** 否。
- **前置依赖：** 模块 4（中文 LLM）+ 模块 5（中文 embedding）。
- **当前是否允许开发：** 不允许。依赖模块 4 和 5 先完成。

### 4. 中文 LLM provider

- **用户价值：** 使用 DeepSeek / Qwen 等国产 LLM，低延迟、低成本、中文能力强。
- **SurfSense 基础能力：** `global_llm_config.example.yaml` 已含 DeepSeek 示例配置（id: -4），支持 DEEPSEEK provider。LiteLLM 已支持 OpenAI 兼容模式，可接入 Qwen。
- **是否配置即可：** 是。填入 API key 即可。
- **是否需要改造：** 低。可能需要调整 token 限制和 temperature 参数。
- **是否需要新增：** 否（Qwen 走 OpenAI 兼容模式即可）。
- **前置依赖：** DeepSeek / Qwen API key。
- **当前是否允许开发：** 配置层面允许（Phase 1 创建 global_llm_config.yaml）。

### 5. 中文 embedding

- **用户价值：** 中文语义检索准确，不因英文 embedding 模型导致中文语义丢失。
- **SurfSense 基础能力：** Chonkie AutoEmbeddings 支持 sentence-transformers 模型。PGVector 支持最多 2000 维。
- **是否配置即可：** 是。`EMBEDDING_MODEL=BAAI/bge-large-zh-v1.5`（1024 维）可直接配置。
- **是否需要改造：** 需要注意：
  - 维度从 384 增加到 1024，存储和检索开销增加约 3x
  - 已有索引需重建（换模型后向量不兼容）
  - chunk_size 可能需要调整（bge-large-zh 的 max_seq_length 为 512）
- **是否需要新增：** 否。
- **前置依赖：** Docker 环境、模型下载（首次启动自动下载约 1.3GB）。
- **当前是否允许开发：** 不允许。需 Phase 1 验证。

### 6. 重点/难点提取

- **用户价值：** 从上传的资料中自动识别核心知识点和难点，帮助学生聚焦学习。
- **SurfSense 基础能力：** 无直接对应功能。但 Agent 架构和 prompt 系统可支撑。
- **是否配置即可：** 否。
- **是否需要改造：** 否（基于现有 Agent 架构）。
- **是否需要新增：** 是。需要新增：
  - 重点提取 prompt 模板
  - 知识点结构化存储（可能利用现有 document/chunk 结构）
  - 前端展示组件
- **前置依赖：** 模块 3（中文问答）正常工作。
- **当前是否允许开发：** 不允许。Phase 2 规划。

### 7. 被提问 / 出题

- **用户价值：** 基于已学资料自动生成练习题，主动检验学习效果。
- **SurfSense 基础能力：** 无直接对应功能。
- **是否配置即可：** 否。
- **是否需要新增：** 是。需要新增：
  - 出题 prompt 模板（选择题/填空题/简答题）
  - 题目结构化 schema
  - 题目存储
  - 前端出题/答题 UI
- **前置依赖：** 模块 6（重点提取）。
- **当前是否允许开发：** 不允许。Phase 2 规划。

### 8. 答题检查

- **用户价值：** 提交答案后获得判定和详细解释。
- **SurfSense 基础能力：** 无。
- **是否需要新增：** 是。需要新增：
  - 答案比对逻辑
  - 解释生成 prompt
  - 评分记录
- **前置依赖：** 模块 7（出题）。
- **当前是否允许开发：** 不允许。Phase 2 规划。

### 9. 错题本

- **用户价值：** 自动收集答错的题目，供集中复习。
- **SurfSense 基础能力：** 无。但 SearchSpace + 文档存储架构可复用。
- **是否需要新增：** 是。需要新增：
  - 错题数据模型
  - 错题列表 API
  - 前端错题本 UI
- **前置依赖：** 模块 8（答题检查）。
- **当前是否允许开发：** 不允许。Phase 2 规划。

### 10. 复习建议

- **用户价值：** 基于遗忘曲线和答题记录，智能推荐复习时间和内容。
- **SurfSense 基础能力：** 无。
- **是否需要新增：** 是。需要新增：
  - 遗忘曲线算法（SM-2 或类似）
  - 复习调度逻辑
  - 前端复习提醒 UI
- **前置依赖：** 模块 8（答题检查）+ 模块 9（错题本）。
- **当前是否允许开发：** 不允许。Phase 3 规划。
